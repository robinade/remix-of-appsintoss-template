# 📋 앱인토스 미니앱 출시 체크리스트

> **이 문서 하나로 앱인토스 출시 완료!**  
> 아래 체크리스트를 순서대로 완료하면 앱인토스에 미니앱을 출시할 수 있습니다.

---

## 🎯 출시까지의 로드맵

```
┌─────────────────────────────────────────────────────────────────────────┐

│ │ 개발 &  │→ │ 앱 아이콘 & │→ │  콘솔   │→ │ 사업자 & │→ │  출시   │ │

```

## ✅ 1단계: 개발 및 테스트 (기술적 요구사항)

### 필수 기술 요구사항

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 1.1 | 라이트 모드만 사용 | 다크모드 CSS 없어야 함 | ☐ |
| 1.2 | 핀치줌 비활성화 | `index.html`에 `user-scalable=no` 설정 | ☐ |
| 1.3 | 커스텀 헤더 없음 | 앱인토스가 네비게이션 바 자동 제공 | ☐ |
| 1.4 | 응답 속도 2초 이내 | 모든 인터랙션 2초 이상 지연 금지 | ☐ |
| 1.5 | 데이터 영구 저장 | 재접속 시 데이터 유지 (네이티브 스토리지 사용) | ☐ |
| 1.6 | 터치 영역 44px+ | 모든 버튼/터치 요소 최소 44px | ☐ |
| 1.7 | 명도 대비 충분 | 텍스트/버튼 대비 확인 | ☐ |
| 1.8 | 외부 링크 금지 | 자사 앱/웹 유도 버튼 없음 | ☐ |

### 로그인 (해당 시)

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 1.9 | 토스 로그인만 사용 | 자사 로그인/기타 간편 로그인 불가 | ☐ |
| 1.10 | 로그인 닫기 동작 | 닫기 시 적절한 화면으로 이동 | ☐ |

### 결제 (해당 시)

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 1.11 | 토스페이만 사용 | 다른 결제 수단 불가 | ☐ |
| 1.12 | 결제 금액 일치 | 주문 요청과 결제창 금액 동일 | ☐ |

### 검증 명령어

```bash
# 빌드 테스트
npm run granite:build

# 샌드박스 앱에서 테스트
npm run granite:dev
```

**✅ 1단계 완료 조건**: 위 항목 모두 체크, 빌드 성공, 샌드박스 테스트 완료

---

## ✅ 2단계: 앱 아이콘 및 브랜딩 준비

### 앱 아이콘 제작

| # | 항목 | 규격 | 확인 |
|---|------|------|:----:|
| 2.1 | 아이콘 크기 | **600x600px** 정사각형 | ☐ |
| 2.2 | 파일 형식 | PNG (투명 배경 불가, 배경색 필수) | ☐ |
| 2.3 | 모서리 형태 | **각진 형태** (둥근 모서리 불가) | ☐ |
| 2.4 | 라이트/다크 대응 | 두 모드에서 모두 잘 보이는 배경색 | ☐ |

> 📥 **아이콘 템플릿 다운로드**:  
> [AppsInToss_Logo_Guide_600_600.pdf](https://developers-apps-in-toss.toss.im/resources/prepare/logo_guide.pdf)

### 아이콘 업로드 위치

```typescript
// granite.config.ts
brand: {
  icon: 'https://your-cdn.com/app-icon-600x600.png', // ← 아이콘 URL 입력
}
```

### 브랜딩 설정

| # | 항목 | 파일 | 확인 |
|---|------|------|:----:|
| 2.5 | 앱 표시 이름 | `granite.config.ts` → `brand.displayName` | ☐ |
| 2.6 | 브랜드 컬러 | `granite.config.ts` → `brand.primaryColor` | ☐ |
| 2.7 | 동일 설정 확인 | `template.config.ts`와 일치 여부 | ☐ |

**✅ 2단계 완료 조건**: 600x600px 아이콘 준비 완료, config 파일 설정 완료

---

## ✅ 3단계: 앱인토스 콘솔 등록

### 콘솔 접속

🔗 **앱인토스 콘솔**: https://console-apps-in-toss.toss.im

### 콘솔 등록 절차

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 3.1 | 회원가입 | 토스 비즈니스 회원으로 가입 (만 19세 이상) | ☐ |
| 3.2 | 워크스페이스 생성 | 팀/프로젝트 이름으로 생성 | ☐ |
| 3.3 | 앱 등록 | '+등록하기' 클릭 | ☐ |

### 앱 기본 정보 입력

| # | 항목 | 입력값 | 확인 |
|---|------|--------|:----:|
| 3.4 | 앱 로고 | 600x600px PNG 업로드 | ☐ |
| 3.5 | 앱 이름 (한글) | 예: "할일 관리" | ☐ |
| 3.6 | appName (영문) | granite.config.ts와 **정확히 일치** 필수! | ☐ |
| 3.7 | 사용 연령 | 현재 만 19세 이상만 지원 | ☐ |
| 3.8 | 고객센터 정보 | 이메일/연락처/채팅 주소 | ☐ |

### ⚠️ 중요: appName 일치 확인

```typescript
// granite.config.ts의 appName과 콘솔의 appName이 정확히 일치해야 합니다!
appName: 'todo-app',  // ← 콘솔에서도 동일하게 'todo-app' 입력
```

**✅ 3단계 완료 조건**: 콘솔에서 앱 등록 완료, 앱 정보 검토 요청 제출

---

## ✅ 4단계: 사업자 등록 및 계약

### 사업자 유형 선택

| 유형 | 수익화 기능 | 토스 로그인 | 필요 서류 |
|------|:----------:|:----------:|----------|
| **사업자 있음** | ✅ 가능 | ✅ 가능 | 사업자등록증 + 추가 서류 |
| **사업자 없음** | ❌ 불가 | ❌ 불가 | 없음 (제한된 기능만 사용) |

### 사업자 등록 절차 (사업자 있는 경우)

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 4.1 | 사업자 정보 입력 | 콘솔 → 파트너 정보 | ☐ |
| 4.2 | 필수 서류 제출 | 사업자등록증 + 대표자 신분증 사본 | ☐ |
| 4.3 | 대표관리자 신청 | 워크스페이스 멤버에서 신청 | ☐ |
| 4.4 | 계약 체결 | 검토 완료 후 계약 진행 | ☐ |

### 사업자가 없는 개인 개발자

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 4.1 | 제한 사항 확인 | 수익화, 토스 로그인 불가 | ☐ |
| 4.2 | 비사업자 출시 신청 | 콘솔에서 별도 절차 진행 | ☐ |

> 💡 **사업자 등록 안내**  
> 사업자가 필요하다면 [국세청 홈택스](https://hometax.go.kr)에서 간편하게 신청 가능 (보통 3일 내 발급)

**✅ 4단계 완료 조건**: 사업자 인증 완료 또는 비사업자 절차 완료, 대표관리자 승인

---

## ✅ 5단계: 출시 요청

### 출시 전 최종 점검

```bash
# 1. 프로덕션 빌드 실행
npm run granite:build

# 2. .ait 파일 생성 확인
ls dist/*.ait
```

### 콘솔에서 출시 요청

| # | 항목 | 설명 | 확인 |
|---|------|------|:----:|
| 5.1 | 앱 번들 업로드 | `.ait` 파일 업로드 | ☐ |
| 5.2 | 앱 버전 입력 | 예: 1.0.0 | ☐ |
| 5.3 | 출시 검토 요청 | '검토 요청하기' 버튼 클릭 | ☐ |
| 5.4 | 검토 결과 확인 | 영업일 1~2일 소요 | ☐ |

### 검토 반려 시

- 콘솔과 이메일로 반려 사유 안내
- 사유 확인 후 수정하여 재요청

**✅ 5단계 완료 조건**: 검토 통과, 출시 완료! 🎉

## 🚨 출시 반려 주요 사유

### 자주 발생하는 반려 사유

| 사유 | 해결 방법 |
|------|----------|
| 커스텀 헤더 사용 | 커스텀 헤더 제거, 앱인토스 네비게이션 바 사용 |
| appName 불일치 | granite.config.ts와 콘솔 appName 일치 확인 |
| 터치 영역 부족 | 모든 버튼 44px 이상으로 수정 |
| 외부 링크 존재 | 자사 앱/웹 유도 버튼 제거 |
| 응답 지연 | 2초 이내 응답하도록 최적화 |
| 다크모드 CSS | 다크모드 관련 스타일 제거 |
